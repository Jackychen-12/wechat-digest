# 公众号推送整理与总结助手

## 为什么会出现抓取 fail / 超时

核心原因通常有三类：

1. **浏览器跨域与反爬限制**：前端页面直接请求 `mp.weixin.qq.com` 或搜索页，容易被 CORS / 风控拦截。
2. **第三方检索源波动**：公众号检索依赖外部页面，目标站点偶发慢响应会导致超时。
3. **本地静态模式缺少 `/api`**：如果只用本地静态服务器，`/api/search-account` 不存在，稳定性会明显下降。

## 本次修复

1. 移除“公众号名与文章链接不匹配”的硬编码保底逻辑。
2. 服务端与前端都改为**多候选并发抓取**，降低单一路径超时导致的整体失败。
3. 前端在静态模式下会给出明确提示：缺少 `/api` 服务，请使用 Vercel 部署地址访问。
4. 公众号检索仍保留链接列表 + 预览 + 入库流程。

## 使用建议

- 文章抓取：优先输入文章链接。
- 公众号抓取：建议使用 Vercel 部署地址访问，成功率高于本地静态模式。

## 部署

推荐 Vercel：导入仓库后直接 Deploy（仓库已包含 `vercel.json` 和 `api/` 目录）。
