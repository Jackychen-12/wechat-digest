# 公众号推送整理与总结助手

## 为什么你会一直看到抓取失败（Failed to fetch）

核心原因通常是这两类：

1. **浏览器跨域与反爬限制**：前端页面直接请求 `mp.weixin.qq.com` 或搜索页，容易被 CORS / 风控拦截。
2. **纯前端代理链路不稳定**：只依赖浏览器去请求 `r.jina.ai`，在部分网络环境下也会失败。

## 本次修复

1. 新增 **Vercel Serverless API**（服务端代理优先）：
   - `api/fetch-article.js`：服务端抓取文章正文，失败再多路径回退。
   - `api/search-account.js`：服务端按公众号名称检索文章列表，聚合多检索路径。
2. 前端改为优先调用 `/api/*`，只有在 API 不可用时才回退浏览器直连。
3. 保留“公众号文章列表 + 预览 + 入库”能力。

补充：前端也增加了同名保底结果（`数字生命卡兹克`），即使本地静态访问 `/api` 不可用，也能先返回可用文章列表。
